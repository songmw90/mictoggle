# Security Policy

## Supported version

Security fixes are applied to the latest code on the `main` branch. No older
release line is currently maintained.

## Reporting a vulnerability

Use GitHub's private vulnerability reporting feature when it is enabled for the
repository. Do not publish credentials, cookies, tokens, or a working exploit in
a public issue. Include affected versions, reproduction steps, expected impact,
and any suggested mitigation.

For ordinary compatibility failures that do not expose data or cross a security
boundary, open a normal issue instead.
